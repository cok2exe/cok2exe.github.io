CSS link 태그를 head 태그 사이에 위치시키면 좋은 이유

  • 페이지가 처음 로드되면 HTML과 CSS가 동시에 파싱됨
  • HTML은 DOM(Document Object Model)을 만들고 CSS는 CSSOM(CSS Object Model)을 만듬
  • 문서 맨 아래에 CSS를 둘 경우, 일부 브라우저(IE 등) 점진적 렌더링을 금지시키게 되므로 사용자는 빈 페이지를 보게 됨

script 태그를 body 태그가 끝나기 직전에 위치시키면 좋은 이유

  • <script> 태그는 다운로드 되고 실행되는 동안 HTML 파싱을 차단하기 때문에,
  • script 태그를 맨 아래에 두면 HTML을 먼저 파싱하여 사용자에게 보여줄 수 있음

img 태그의 srcset 속성

  • 기기 디스플레이 너비에 따라 다른 이미지를 보여줄 때 사용됨
  • 데이터 낭비를 줄일 수 있게 됨 (저해상도 이미지는 저사양 기기에, 고해상도 이미지는 고사양 기기에)

🦄 CSS

Resetting 과 Normalizing CSS의 차이점

  • Resetting 모든 기본 브라우저 스타일을 제거
  • Normalizing 필요한 일부 스타일을 남기고 제거 (나는 Normalizing을 활용하는 편)

float

  • CSS의 위치지정 속성
  • float된 요소는 페이지 흐름의 일부가 됨
  • 페이지 흐름에서 제거되는 position: absolute 와는 다름
  • 부모요소에 float 요소가 있으면 높이가 무효화됨

무효화된 높이의 해결 방법

  • CSS의 가상 선택자 :after를 사용하여 float을 clear 해주면 해결됨

  • 부모 요소에 overflow: auto; 또는 overflow: hidden;

  • 부모 요소 다음 라인에 <div style="clear: both;"></div> 추가

CSS 스프라이트 (CSS sprite)

  • 여러 이미지를 하나의 큰 이미지로 결합
  • 일반적으로 아이콘에 사용되는 기술

장점

  • 여러 이미지에 대한 HTTP 요청 수를 줄임
  • 여러개의 아이콘을 한페이지에 보여줘야 하는 경우, 하나씩 뜨는 문제 해결
  • CSS :hover 시 아이콘이 변경될 때, (hover용 이미지를 다운받기 위해) 깜빡이는 문제 해결

CSS 전처리기의 장단점

장점

  • CSS의 유지보수 향상
  • 중첩 선택자 사용하기 쉬움
  • 변수 사용 가능
  • 반복되는 CSS를 위한 Mixins 생성 (재사용성 증가)

단점

  • 전처리기를 위한 도구 필요 (컴파일시 시간 증가)

Pseudo-elements

  • clearfix에 사용
  • 말풍선의 삼각형 그릴 때 사용
  • :nth-child(xn) (float 사용시 clear: both; 해줘야 할때 사용) 또는 줄무늬 그려줄때?

box-sizing: boder-box

  • border와 padding도 width, height 계산에 사용됨
  • 디자이너가 의도한 사이즈와 일치하게 적용 가능

position

  • static: 기본 위치. top, right, left, bottom, z-index 적용 안됨.
  • relative: 요소의 위치가 레이아웃을 변경하지 않고 자체에 상대적으로 조정됨.
  • absolute: 요소의 위치가 평소의 위치에서 제거됨. 가장 가까운 relative 부모의 블록에 의존. (그렇지 않으면 최상위 블록에 의존). absolute에 margin이 주어지면 다른 margin과 충돌하지 않음. 이 요소는 다른 요소에 영향을 주지 않음.
  • fixed: 요소의 위치가 평소의 위치에서 제거됨. 뷰포트를 기준으로 위치가 지정되며 스크롤해도 해당 위치에 배치되어 있음.
  • sticky: relative와 fixed의 하이브리드. 요소는 지정된 임계값을 넘을 때까지 relative 위치로 처리되며, 특정 지점에서 fixed 위치로 처리됨.
