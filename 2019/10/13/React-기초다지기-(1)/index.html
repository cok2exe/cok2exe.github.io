React 기초다지기 (1)

2019/10/13 카테고리 React

리액트를 실무에서 아무런 기초없이 맞닥뜨리고 사용하게 되면서 어찌저찌 리액트에 대해 알게 되었지만, 기초부터 다시 다져보자는 마인드로 책을 펴서 공부를 시작하게 되었다.

이미 알고 있는 내용이지만 좀 더 자세한 설명과 함께 한 번 더 알아갈 수 있어 더 좋은 것 같다.

React?

오직 View만 신경쓰는 라이브러리 (프레임 워크 아님)

Virtual DOM

DOM (Document Object Model)의 복사본

<리액트에서의 Virtual DOM의 과정>

  1. 데이터 업데이트 => 전체 UI를 Virtual DOM에 리렌더링
  2. 이전 Virtual DOM과 현재 Virtual DOM을 비교
  3. 바뀐 부분만 실제 DOM에 적용

JSX

  • 자바스크립트의 확장 문법으로 XML과 매우 비슷
  • 브라우저에서 실행되기 전에 코드가 번들링되는 과정에서 babel을 사용하여 일반 자바스크립트 형태의 코드로 변환

JSX 장점

  1. 보기 쉽고 익숙하다.
  2. 더욱 높은 활용도

JSX 문법

  1. 무조건 부모 요소로 감싸야함
    => why? Virtual DOM에서 컴포넌트 변화를 감지할 때, 효율적으로 비교할 수 있도록 컴포넌트 내부는 하나의 DOM 트리 구조로 이루어져야 하는 규칙이 있기 때문
  2. JSX 안에서는 자바스크립트 표현식을 쓸 수 있음 (ex. {name})
  3. if문 대신 조건부 연산자(=삼항 연산자) 사용 가능
  4. AND 연산자(&&)를 사용한 조건부 렌더링 가능
  5. undefined를 렌더링하지 않기
  6. 인라인 스타일링은 객체 형태로 넣어주어야 하며, camelCase를 사용한다.
  7. class => className
  8. 태그는 꼭 닫혀야 함 (ex. <input> XXX => <input /> OOO)
  9. 주석은 { /* ... */ } 이렇게!

컴포넌트

클래스형 컴포넌트

  • state 기능 및 life-cycle API 사용 가능
  • 임의 메서드 정의 가능

vs. 함수형 컴포넌트 시 함수형 컴포넌트의 장점

vs. 함수형 컴포넌트 시 함수형 컴포넌트의 단점

props

부모 컴포넌트에서 설정함

  • 기본 값 설정 시엔 defaultProps
  • props.children => 컴포넌트 태그 사이의 내용을 보여주는 props
  • propTypes를 통한 props 검증 + isRequired를 이용하여 필수 propTypes 설정 가능

state

컴포넌트 내부에서 바뀔 수 있는 값

  • 하나의 함수에서 this.setState를 두 번 호출하면, 바로 바뀌지 않기 때문에 원하는 결과값을 얻을 수 없음
    => 해결법: this.setState(prevState => …)
